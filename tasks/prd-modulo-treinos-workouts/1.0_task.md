## Detalhes de Implementação

Consultar `techspec.md` → seção "Interfaces Principais" e "Modelos de Dados".

- Usar `useSupabaseUser()` para obter user_id em todas as queries
- Transação para createWorkout: insert workout → pegar id → bulk insert workout_exercises
- updateWorkout: update workout → delete exercises antigos → upsert novos
- logSet: upsert baseado em session_id + workout_exercise_id + set_number

## Critérios de Sucesso

- Todos os métodos retornam dados ou erro corretamente
- createWorkout cria workout + exercises em transação
- logSet salva incrementalmente sem perda de dados
- Toast de erro em falhas de Supabase
- Tipos TypeScript sem erros (npx nuxi typecheck)
