## Visão Geral

Aplicar a proteção de roles em todas as páginas existentes do FitPulse usando `definePageMeta` com `requiredRoles`, e executar testes E2E completos com Playwright para validar todo o fluxo de autenticação + autorização + isolamento de dados. Esta é a tarefa de integração final que conecta todas as peças.

<requirements>

### RF-07
Cada rota protegida deve declarar explicitamente quais roles têm acesso permitido

### RF-10
Rotas públicas (landing page, login, registro) não devem exigir validação de role

### RF-17
Um Personal Trainer deve visualizar apenas dados dos alunos vinculados a ele

### RF-18
Dados corporais e de saúde de um usuário devem ser visíveis apenas por ele e seu PT vinculado

### RF-19
O Superadmin deve ter acesso de leitura a todos os dados da plataforma para fins de auditoria

### RF-20
O sistema deve impedir que um usuário acesse dados de outro usuário sem vínculo explícito

</requirements>
