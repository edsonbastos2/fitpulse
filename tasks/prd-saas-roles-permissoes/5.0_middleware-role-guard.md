# Tarefa 5.0: Middleware `role-guard`

<critical>Ler os arquivos de prd.md e techspec.md nesta pasta. Se você não ler esses arquivos, sua tarefa será invalidada.</critical>

## Visão Geral

Criar o middleware Nuxt `role-guard.ts` que valida se o usuário possui os roles necessários para acessar cada rota. Este middleware funciona em cadeia com o `auth.ts`: primeiro o `auth` verifica se o usuário está logado, depois o `role-guard` verifica se ele tem permissão de role para aquela rota.

## Subtarefas

- [ ] 5.1 Criar `middleware/role-guard.ts` seguindo o design da techspec
- [ ] 5.2 Ler `to.meta.requiredRoles` da rota de destino
- [ ] 5.3 Se `requiredRoles` está definido e não vazio, verificar com `useRoles().hasAnyRole()`

## Detalhes de Implementação

Consultar a **Tech Spec** nas seções:
- **Middleware de Rotas** → código completo de `middleware/role-guard.ts`

O middleware deve ser simples — ~15 linhas. A lógica complexa fica no `useRoles`. Uso de `defineNuxtRouteMiddleware` padrão do Nuxt 3.

## Critérios de Sucesso

- [ ] Middleware `role-guard.ts` existe e é registrado automaticamente pelo Nuxt
- [ ] Rota com `requiredRoles: ['user']` permite acesso a usuários com role `user`
- [ ] Rota com `requiredRoles: ['personal_trainer']` nega acesso a usuários com apenas role `user`
- [ ] Acesso negado redireciona para `/access-denied?required=personal_trainer&redirect=/dashboard`
- [ ] Rota sem `requiredRoles` permite acesso a qualquer usuário autenticado
- [ ] Log `console.warn` é emitido em cada tentativa de acesso negado
- [ ] Middleware roda após `auth.ts` na cadeia (usuário não autenticado é barrado antes de verificar role)
