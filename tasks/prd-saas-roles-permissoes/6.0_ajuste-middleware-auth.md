## Visão Geral

Simplificar o middleware `auth.ts` existente para que cuide exclusivamente de autenticação (verificar se usuário está logado), removendo a lógica de validação de roles que agora é responsabilidade do `role-guard.ts`. Esta tarefa é simples mas crítica — garante separação limpa de responsabilidades entre auth e autorização.

Consultar a **Tech Spec** nas seções:

O middleware após ajuste deve ficar com ~10 linhas:
```ts
export default defineNuxtRouteMiddleware((to) => {
  const user = useSupabaseUser()
  
  const protectedRoutes = ['/dashboard', '/workouts', '/exercises', '/profile', '/settings', '/schedule', '/progress']
  const isProtectedRoute = protectedRoutes.some((route) => to.path.startsWith(route))
  
  if (isProtectedRoute && !user.value) {
    return navigateTo('/auth/login')
  }
  
  const authRoutes = ['/auth/login', '/auth/register']
  const isAuthRoute = authRoutes.includes(to.path)
  
  if (isAuthRoute && user.value) {
    return navigateTo('/dashboard')
  }
})
```
