# Tarefa 7.0: Página `access-denied`

<critical>Ler os arquivos de prd.md e techspec.md nesta pasta. Se você não ler esses arquivos, sua tarefa será invalidada.</critical>

## Visão Geral

Criar a página `pages/access-denied.vue` que informa ao usuário que ele não tem permissão para acessar determinado recurso, explica o motivo de forma clara e oferece caminhos de ação (upgrade de role, voltar para dashboard). A página deve seguir o design system existente do FitPulse (tema escuro, gradientes, glassmorphism) e ser acessível.

- **`frontend-design`** — UI consistente com design system existente do FitPulse
- **`web-design-guidelines`** — Acessibilidade, navegação por teclado, screen readers
</skills>

## Detalhes de Implementação

Consultar a **Tech Spec** nas seções:
- **Middleware de Rotas** → query params passados no redirect (`required`, `redirect`)
- **Considerações de UI/UX** no PRD → "informativa, não punitiva"

A página deve ser simples — um card centralizado com:
- Ícone de alerta (ex: `ExclamationTriangleIcon` do Heroicons)
- Título: "Acesso Restrito"
- Descrição explicando o que é necessário
- Badges mostrando os roles necessários
- Botões de ação (upgrade + voltar)

Usar `UiCard` existente para consistência.
