# Tarefa 8.0: Componente `RoleSwitcher`

<critical>Ler os arquivos de prd.md e techspec.md nesta pasta. Se você não ler esses arquivos, sua tarefa será invalidada.</critical>

## Visão Geral

Criar o componente `components/layout/RoleSwitcher.vue` que permite usuários com múltiplos roles alternar entre contextos de uso (ex: Usuário Final ↔ Personal Trainer). O componente deve ser exibido no header/sidebar quando o usuário possui mais de um role, usando um dropdown ou toggle visual com badges de identificação.

## Detalhes de Implementação

Consultar a **Tech Spec** nas seções:
- **Interfaces Principais** → `useRoles()` com `activeRole` e `setActiveRole`
- **Considerações de UI/UX** no PRD → "Indicador visual do perfil ativo (badge/tag)"

O componente deve ser compacto — um toggle horizontal ou dropdown pequeno posicionado no header ou sidebar. Exemplo visual:

```
Visualizando como: [🟢 Usuário] [🔵 Personal Trainer]
                            ^^^^^ ativo
```

Usar `UiBadge` existente para os roles.

## Critérios de Sucesso

- [ ] Componente só aparece quando usuário tem 2+ roles
- [ ] Cada role é exibido como badge clicável com cor distinta
- [ ] Role ativo tem destaque visual (borda, preenchimento, checkmark)
- [ ] Ao clicar em outro role, `activeRole` muda e UI reflete imediatamente
- [ ] Componente é responsivo (funciona em mobile e desktop)
- [ ] Acessível: `role="radiogroup"` no container, `role="radio"` nos itens
- [ ] Label "Visualizando como:" é visível e clara
- [ ] Zero renderização quando usuário tem apenas 1 role
