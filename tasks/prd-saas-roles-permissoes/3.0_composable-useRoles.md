# Tarefa 3.0: Composable `useRoles`

<critical>Ler os arquivos de prd.md e techspec.md nesta pasta. Se você não ler esses arquivos, sua tarefa será invalidada.</critical>

## Visão Geral

Criar o composable Vue `useRoles` que fornece verificação reativa de roles e permissões no frontend. Este é o coração da lógica de autorização no lado cliente — consumido pelo middleware `role-guard`, pela página `access-denied`, pelo componente `RoleSwitcher` e por qualquer componente que precise renderização condicional por role.

## Detalhes de Implementação

Consultar a **Tech Spec** nas seções:
- **Interfaces Principais** → código de exemplo de `composables/useRoles.ts`

O composable deve usar `useState` (não `ref`) para `roles` e `activeRole` para que o estado seja compartilhado entre todos os componentes e middlewares no mesmo request/contexto Nuxt.

## Critérios de Sucesso

- [ ] `useRoles()` exporta `roles`, `hasRole`, `hasAnyRole`, `can`, `activeRole`, `setActiveRole`
- [ ] `hasRole('user')` retorna `true` para usuário com role `user`
- [ ] `hasRole('personal_trainer')` retorna `false` para usuário que só tem role `user`
- [ ] `hasAnyRole(['personal_trainer', 'superadmin'])` retorna `true` se usuário tem qualquer um dos dois
- [ ] `can('view_students')` retorna `true` para `personal_trainer`, `false` para `user`
- [ ] `setActiveRole('personal_trainer')` só funciona se usuário possui esse role
- [ ] `setActiveRole('superadmin')` não funciona para usuário sem esse role (ignora silently)
- [ ] Estado `roles` é compartilhado entre componentes (mesmo `useState` key)
