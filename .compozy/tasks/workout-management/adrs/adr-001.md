# ADR-001: Escopo V1 — Core Workout Management

**Status:** Accepted  
**Date:** 2026-04-13  
**Context:** Idea Factory — Workout Management Module

## Decision

Focar a V1 no core workout management: criação de treinos (livre + templates básicos), execução com logging simplificado (peso, reps, timer de descanso), e histórico de sessões com feedback.

## Problem

O FitPulse não possui funcionalidade de gestão de treinos — o botão "Novo Treino" no dashboard leva a uma rota 404. Sem este módulo, o app é apenas um catálogo de exercícios sem utilidade prática.

## Alternatives Considered

### A) CRUD mínimo sem templates
- **Pró:** Entrega mais rápida, escopo mínimo
- **Contra:** Alta fricção de onboarding — usuários sem referência abandonam (40% churn por complexidade)
- **Resultado:** Rejeitado

### B) Full feature com analytics e progresso
- **Pró:** Produto completo desde dia 1
- **Contra:** Complexidade alta, risco de entrega, sem dados para validar value proposition
- **Resultado:** Rejeitado — analytics dependem de dados que ainda não existem

### C) Escopo escolhido: Core + Templates básicos + Feedback
- **Pró:** Valida core value, reduz fricção, entrega rápido
- **Contra:** Requer V2 para analytics que usuários esperam
- **Resultado:** Aceito

## Consequences

### Positive
- Entrega mais rápida ao mercado (estimativa reduzida vs full feature)
- Valida core value proposition antes de investir em features avançadas
- Gera dados de uso reais para informar V2 (analytics, progresso)
- Templates básicos resolvem problema de "cold start" para novos usuários

### Negative
- Usuários avançados podem querer gráficos de progresso imediatamente
- Sem agendamento na V1 — funcionalidade existente no schema ficará ociosa
- Offline mode ausente pode limitar uso em academias com sinal ruim

### Risks
- Logging precisa ser < 30s/set para evitar churn — UI/UX é crítico
- Estado de sessão em progresso precisa sobreviver a reload de página
